# Roadmap — Interview Helper

Тренажёр для подготовки к собеседованиям. Старт: позиция System Analyst Junior.
Долгосрочное видение: формат в духе Duolingo (привычка, streak, геймификация),
но для технических собеседований.

> Принцип этого roadmap: сначала проверяем, что продукт нужен людям,
> и только потом вкладываемся в инфраструктуру. Ориентир нагрузки — до 200
> пользователей в день. Это сознательно удерживает от преждевременного усложнения.

---

## Фаза 0 — Фундамент (сделано)

- [x] Концепт и выбор стека (Flutter — iOS + Android из одного кода)
- [x] Структура проекта
- [x] MVP: экраны (главный, сессия, результат)
- [x] Вопросы с вариантами ответа + мгновенный фидбэк
- [] Локальное хранение прогресса (XP, streak, прогресс по темам)
- [x] Банк вопросов в JSON (4 темы, базовый набор)

---

## Фаза 1 — Довести MVP до состояния «можно дать людям»

**Цель фазы:** 5–10 реальных людей проходят тренажёр и говорят, полезно или нет.
**Контент здесь важнее кода.**

- [ ] Наполнить банк вопросов: 30–50 вопросов на каждую тему
- [ ] Добавить темы: SQL-джойны, методологии (Agile/Scrum), поведенческие (STAR)
- [ ] Экран «разбор ошибок» после сессии (показывать, где ошибся, с пояснением)
- [ ] Причесать UX: пустые состояния, анимации переходов, обработка краёв
- [ ] Иконка приложения и название
- [ ] Собрать обратную связь от первых пользователей

**Критерий перехода к Фазе 2:** есть подтверждение, что формат полезен.

---

## Фаза 2 — Бэкенд и подключение Claude

**Цель фазы:** обновлять контент без релиза + оценивать открытые ответы.
Здесь впервые задействуется собственный сервер.

- [ ] Эндпоинт на сервере, отдающий банк вопросов (`GET /questions`)
- [ ] Переключить приложение на загрузку вопросов с сервера (пакет `http` уже есть)
- [ ] Кэширование вопросов на устройстве (работа офлайн + скорость)
- [ ] Прокси-эндпоинт для Claude (`POST /evaluate`) — ключ хранится ТОЛЬКО на сервере
- [ ] Открытые вопросы: пользователь пишет ответ → сервер → Claude → оценка
- [ ] Базовая защита эндпоинтов (rate limiting, простая авторизация)

**Важно по безопасности:** API-ключ Anthropic никогда не попадает в приложение.
Схема всегда: приложение → сервер → Claude API → сервер → приложение.

## Фаза 3 — Удержание (превращаем заходы в привычку)

**Цель фазы:** пользователь возвращается регулярно.

- [ ] Аккаунты (регистрация / вход)
- [ ] Синхронизация прогресса между устройствами
- [ ] Push-уведомления («не теряй streak», напоминание о дневной цели)
- [ ] Дневные цели и достижения
- [ ] Статистика прогресса по темам (графики, слабые места)

---

## Фаза 4 — Рост и контент

**Цель фазы:** расширение за пределы одной позиции.

- [ ] Новые позиции (Business Analyst, QA, Backend Junior и т.д.)
- [ ] Новые форматы вопросов (сценарии, кейсы, «найди ошибку»)
- [ ] Возможные соцфичи (лиги, рейтинги) — по примеру Duolingo
- [ ] Публикация: Android (Google Play), затем iOS (нужен Mac + Apple Developer Account, 99 $/год)

## Технические заметки

- **Стек:** Flutter (Dart). Один код под iOS и Android.
- **Хранение (сейчас):** локально на устройстве через `shared_preferences`.
- **Сервер:** появляется в Фазе 2, не раньше.
- **Масштаб:** проектируем под ~200 DAU. Не строим инфраструктуру «на вырост»
  раньше времени — это можно усложнить позже, когда появится реальная нагрузка.

## Принципы принятия решений

1. Не строим то, что ещё не нужно (YAGNI).
2. Контент и проверка гипотезы важнее технического совершенства.
3. Каждая фаза имеет смысл только если предыдущая дала результат.
4. Простое решение, которое работает сегодня, лучше идеального через полгода.
