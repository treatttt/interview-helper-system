# Interview Helper — тренажёр собеседований (System Analyst Junior)

Кроссплатформенное приложение на Flutter (iOS + Android из одного кода).
MVP: вопросы с вариантами ответа, мгновенный фидбэк, прогресс по темам, XP и streak.

---

## Что это и чего тут пока нет

Это **рабочий прототип MVP**. Сейчас всё работает локально на устройстве:
банк вопросов лежит в `assets/data/questions.json`, прогресс хранится на телефоне.
Сервер и оценка открытых ответов — следующий этап (см. ниже).

---

## Как запустить (пошагово, ты не писал на Flutter — это нормально)

### Шаг 1. Установить Flutter SDK
1. Открой https://docs.flutter.dev/get-started/install
2. Выбери свою ОС (Windows / macOS / Linux) и пройди установку.
3. В терминале проверь: `flutter doctor`
   Он покажет, чего не хватает. Зелёные галочки — хорошо; на красное он сам подскажет, что доустановить.

### Шаг 2. Положить файлы проекта
Распакуй архив. Внутри уже готовая структура (`lib/`, `assets/`, `pubspec.yaml`).
Положи содержимое в свой репозиторий `interview-helper-system`.

### Шаг 3. Подтянуть зависимости
В папке проекта:
```bash
flutter pub get
```

### Шаг 4. Запустить
**Проще всего — Android-эмулятор или браузер (Mac не нужен):**
```bash
flutter run            # выберет доступное устройство
flutter run -d chrome  # запустить как веб-страницу прямо в браузере
```
Веб-запуск (`-d chrome`) — самый быстрый способ просто посмотреть, как всё работает.

**Android на реальном телефоне:** включи на телефоне «Режим разработчика» + «Отладку по USB», подключи кабелем, запусти `flutter run`.

**iOS:** для сборки под iPhone нужен **Mac с Xcode**. Это ограничение Apple — обойти нельзя. На Windows/Linux iOS-версию собрать не получится, но Android и веб дадут тебе ровно ту же функциональность для теста.


## Как добавлять вопросы

Открой `assets/data/questions.json` и добавь объект в `questions` нужной темы:
```json
{
  "id": "db5",
  "text": "Текст вопроса?",
  "options": ["Вариант 1", "Вариант 2", "Вариант 3", "Вариант 4"],
  "correct": 0,
  "explanation": "Почему правильный ответ именно такой."
}
```
`correct` — индекс правильного варианта (счёт с 0). После правки перезапусти приложение.

Контент важнее кода — на этом этапе самое полезное, что можно делать, это наполнять банк вопросов.

## Подключение твоего сервера (следующий этап)

Сейчас вопросы грузятся из локального файла. Чтобы брать их с сервера:
1. Подними на сервере эндпоинт, отдающий тот же JSON (например `GET /questions`).
2. В `services/question_repository.dart` замени чтение ассета на HTTP-запрос
   (пакет `http` уже подключён в `pubspec.yaml`).

Это же место — будущая точка для оценки открытых ответов: запрос на сервер,
сервер обращается к LLM, возвращает оценку. Так ключ от LLM не попадёт в приложение.
